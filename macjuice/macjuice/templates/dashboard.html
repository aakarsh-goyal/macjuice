⚡ macjuice manual

What it is

A free, self-hosted battery monitor for your Mac. It shows everything coconutBattery and Battery Health charge money for — long-term history graphs, charge/discharge rates, runtime estimates, battery-health trend, per-session breakdowns — using only macOS's own built-in tools. No paid app, no cloud, no admin rights.

How it works

Two pieces share one small database:

  • The collector — a tiny background agent (macOS launchd) that wakes every 120 seconds, reads your battery via ioreg + pmset (and system_profiler once an hour), and appends one row to the database. It runs whether or not this page is open, and restarts at login.
  • This dashboard — a read-only local web server that reads that database and draws what you see. It only needs to run when you want to look.

It's gentle on battery: the cheap reads take ~30 ms; the heavy hourly read is cached. It takes no power assertion, so it never wakes your Mac from sleep — you just won't get samples while it's asleep, which is fine.

Reading the screen

  • Gauge — current charge. Colour follows state: teal = charging, amber = discharging, red = critical.
  • Power draw — live watts in/out (↓ draining, ↑ charging) plus the recent %/hour rate.
  • Two health numbers — mAh is the raw max÷design capacity (can read >100% on a fresh battery, like coconutBattery); Apple is the smoothed % macOS shows in Settings. They legitimately differ.
  • Est. full runtime — how long a full 100% would last at the recent drain rate. "Short" reacts fast; "medium" is steadier.
  • Sessions — each unplugged stretch from plug-out to plug-in, with how much you drained.
  • History — pick 24H/7D/30D/ALL. Older points before install are coarse "backfill" estimated from system logs; detail sharpens after install.

A blank value ("—") just means that reading isn't in yet — e.g. condition/Apple-health populate on the collector's first hourly cycle.

Turn the dashboard off / on

This only closes the viewer. The collector keeps logging in the background.

stop the dashboard
pkill -f macjuice.app
start it again, then open localhost:5137
cd ~/Projects/macjuice && MACJUICE_DB="$HOME/Library/Application Support/macjuice/battery.db" PYTHONPATH=macjuice macjuice/.venv/bin/python -m macjuice.app

Pause / resume background logging

check it's running
launchctl list | grep macjuice
pause logging (temporary)
launchctl unload ~/Library/LaunchAgents/com.macjuice.collector.plist
resume logging
launchctl load ~/Library/LaunchAgents/com.macjuice.collector.plist

Uninstall

Removes the background agent. Your recorded data is kept unless you delete it.

remove the agent
cd ~/Projects/macjuice && ./uninstall.sh
also delete all recorded data (optional)
rm -rf "$HOME/Library/Application Support/macjuice"

Reinstall / update

reinstall (safe to re-run anytime)
cd ~/Projects/macjuice && ./install.sh
pull the latest version from GitHub
cd ~/Projects/macjuice && git pull && ./install.sh

Where things live

  • Code — ~/Projects/macjuice
  • Database — ~/Library/Application Support/macjuice/battery.db
  • Logs — ~/Library/Logs/macjuice/
  • Agent — ~/Library/LaunchAgents/com.macjuice.collector.plist
  • Repo — github.com/raghavtripped/macjuice

Troubleshooting

Page won't load? The dashboard server isn't running — start it with the command under "Turn the dashboard off / on".

Charts look empty or flat? The collector needs a little runtime to gather live points — give it a few minutes, or check it's loaded with launchctl list | grep macjuice. The status pill (top-right) turns red when it can't reach the server.
